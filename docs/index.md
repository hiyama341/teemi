# teemi

<div class="hero">
  <div class="hero-copy">
    <p class="hero-kicker">Scientific Python toolkit</p>
    <h1>Reproducible design-build-test-learn workflows for microbial strain engineering</h1>
    <p class="hero-lead">
      <code>teemi</code> helps scientists and engineers connect sequence design, cloning workflows,
      laboratory tracking, and downstream analysis in notebook-friendly pipelines.
    </p>
  </div>
</div>

The project is for researchers who want a practical Python toolkit for moving from biological design ideas to
documented build plans, test workflows, and analysis outputs in one place.

<p class="fair-callout">
  The name <code>teemi</code> comes from the Greek goddess of fairness, reflecting the project's goal of making
  bioengineering more FAIR: findable, accessible, interoperable, and reusable.
</p>

## What problem teemi solves

`teemi` is most useful when a workflow spans several stages of strain construction and you want the same codebase
to support design decisions, experiment planning, sample tracking, and downstream analysis.

## Citation

If you use teemi in scientific work, cite the project publication:
